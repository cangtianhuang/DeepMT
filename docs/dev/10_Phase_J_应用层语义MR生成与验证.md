# 10_Phase_J_应用层语义MR生成与验证.md

## 1. 阶段目标

本阶段目标是实现应用层语义 MR 的最小可行闭环，让论文中“基于文档知识与大语言模型的应用层关系生成”从概念走向可答辩、可实验的系统能力。

阶段完成后，DeepMT 应至少具备以下能力：

- 针对少量代表性应用场景构建知识检索与上下文整理流程；
- 借助 LLM 生成应用层语义 MR 候选；
- 对候选关系进行规范化和可执行化处理；
- 在有限场景中执行验证并记录质量结果；
- 将有效关系纳入统一知识库。

---

## 2. 本阶段要解决的问题

应用层是论文中最具创新性但当前也最空缺的部分。若完全不落地，论文完整性会受到影响；若做得过宽，又极易失控。

因此，本阶段必须坚持“小范围、强约束、可验证”的开发原则，只做能支撑论文叙述和实验展示的最小可行实现。

---

## 3. 阶段边界

本阶段建议只覆盖两个代表性应用方向，例如：

- 图像分类类应用；
- 文本分类/情感分析类应用。

编码智能体可结合现有资源自主微调场景，但必须保证：

- 场景数少；
- 关系语义清晰；
- 验证流程可控；
- 结果可以进入统一仓库与实验统计。

本阶段不做的内容包括：

- 不构建大而全的多领域知识图谱平台；
- 不追求复杂对话式多智能体系统；
- 不在本阶段覆盖大量开放任务；
- 不把应用层做成不可验证的“自然语言想法集合”。

---

## 4. 任务拆分

### J1：应用场景选定与样例集准备

目标：先收缩问题边界，明确应用层要测试的具体任务与样例。

本任务需要完成：

- 选定少量代表性应用场景；
- 整理样例输入、模型或服务入口；
- 明确每类场景的输出形式；
- 为后续语义变换设计提供基础样本；
- 形成应用层 benchmark 清单。

### J2：知识检索与上下文构建器

目标：让应用层 MR 生成建立在可追踪的知识依据上，而不是只靠 LLM 自由发挥。

本任务需要完成：

- 复用现有 web search / agent 工具能力；
- 为应用场景构建检索查询；
- 抽取与场景相关的文档片段；
- 组织成适合提示生成的结构化上下文；
- 保留文档来源与上下文快照，便于追踪和人工复核。

### J3：应用层提示模板与候选 MR 生成

目标：建立受控的应用层语义 MR 候选生成流程。

本任务需要完成：

- 设计应用层 prompt 模板；
- 约束 LLM 输出格式；
- 生成候选语义 MR；
- 对候选进行去重、粗筛和异常格式处理；
- 保留 prompt 版本与生成来源。

### J4：语义 MR 规范化与可执行化

目标：把自然语言候选转换为系统可管理、可验证的关系记录。

本任务需要完成：

- 设计应用层关系卡片或规范化结构；
- 将语义描述映射为变换规则、适用边界和验证协议；
- 标记高风险或低确定性关系；
- 与统一 repository 对接；
- 保证后续验证模块可以直接消费这些关系。

### J5：应用层验证与质量筛选

目标：建立应用层关系的基础质量保障流程。

本任务需要完成：

- 设计适用于应用层的预筛选策略；
- 支持少量样例上的试运行验证；
- 处理语义保持不完全严格的情况；
- 输出通过、失败、待人工确认等状态；
- 将结果反馈到关系生命周期管理中。

### J6：人工复核与入库闭环

目标：避免应用层关系全部依赖自动判断，建立轻量而必要的人工复核入口。

本任务需要完成：

- 设计人工复核视图或导出格式；
- 支持查看关系来源、上下文依据和验证结果；
- 允许人工确认、驳回或标注风险；
- 将复核结果写回关系状态；
- 让入库关系具备可追踪审核痕迹。

### J7：应用层结果展示与样例报告

目标：让应用层能力可以被论文和答辩直接展示。

本任务需要完成：

- 输出应用层关系数量与质量统计；
- 输出少量典型关系样例；
- 输出验证通过和失败的案例说明；
- 为实验组织器和只读仪表盘提供数据接口；
- 形成应用层最小展示链路。

本阶段建议优先新增或调整以下区域：

- `deepmt/application/`

## 6. 可运行成果

本阶段结束后，系统至少应能做到：

1. 在少量应用场景上完成知识检索、候选生成、规范化、验证和入库；
2. 输出一批应用层语义 MR；
3. 为每条应用层关系保留来源与验证记录；
4. 形成可展示的应用层样例报告。

---

## 7. 验收标准

本阶段验收重点是“应用层是否形成可控、可追踪、可展示的最小闭环”：

- 至少两个应用方向中的一个完整跑通；
- 应用层关系可规范化存储并带有来源信息；
- 有基础验证与人工复核能力；
- 有少量可直接写入论文或演示的应用层案例；
- 文档清楚说明适用范围、风险边界与后续扩展位。

---

## 8. 与论文章节的对应关系

本阶段主要支撑以下论文内容：

- 第 3.4 节：应用层蜕变关系自动生成方法；
- 第 4.2.3 节：应用层生成引擎实现；
- 第 5.2 节：蜕变关系生成效果评估中的应用层部分。

---

## 9. 开发完成标志

当以下条件同时满足时，可认为 Phase J 完成：

- 应用层已经有真实可运行的生成与验证链路；
- 应用层 MR 不再停留在理论描述；
- 系统可以展示来源可追踪、质量可解释的应用层关系；
- 论文与答辩中可引用至少一组应用层示例。
