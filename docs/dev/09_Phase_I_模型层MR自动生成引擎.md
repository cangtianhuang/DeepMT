# 09_Phase_I_模型层MR自动生成引擎.md

## 1. 阶段目标

本阶段目标是把“模型层”从论文中的方法描述，真正落地为系统中的可运行能力。

阶段完成后，DeepMT 应至少具备以下能力：

- 能够识别和加载一批代表性模型；
- 能够对模型结构进行基础分析；
- 能够针对模型特性自动生成一批模型层 MR；
- 能够执行模型层蜕变测试并记录结果；
- 能够输出模型层失败案例和统计摘要。

---

## 2. 本阶段要解决的问题

当前系统已经在算子层形成较强闭环，但模型层仍主要停留在论文设计和预留接口层面。若模型层不落地，论文中的“分层蜕变测试体系”会长期停留在一层半状态，难以形成与题目一致的系统事实。

因此，本阶段是后续开发中最关键的研究兑现阶段之一。

---

## 3. 阶段边界

本阶段遵循“先做小而硬的模型层闭环，再逐步扩展覆盖面”的原则。

本阶段建议优先覆盖以下对象：

- 简单 CNN；
- 简单 MLP；
- 小型 RNN / LSTM；
- 小型 Transformer encoder 类模型。

本阶段不做的内容包括：

- 不以大规模预训练模型为首批对象；
- 不追求复杂分布式训练支持；
- 不做过多与任务无关的模型可视化界面；
- 不一次性覆盖全部模型结构模式。

---

## 4. 任务拆分

### I1：模型对象接入与基准集整理

目标：先定义和整理一批可稳定运行的模型层测试对象。

本任务需要完成：

- 建立模型层 benchmark 清单；
- 明确模型来源、加载方式和输入规格；
- 划分推理型与训练型模型场景；
- 保证每个模型都能在统一入口下被加载和执行；
- 为后续结构分析和 MR 生成提供规范输入。

### I2：模型结构分析器第一版

目标：让系统具备基础的模型图结构识别能力。

本任务需要完成：

- 建立模型结构抽取流程；
- 输出基础层类型序列、连接关系和关键子结构摘要；
- 识别对 MR 生成有价值的组件模式；
- 将分析结果转为后续生成器可消费的结构信息；
- 为模型层日志和调试保留可读输出。

### I3：模型层变换策略库

目标：建立模型层可复用的输入或结构变换策略集合。

本任务需要完成：

- 梳理适用于模型层的常见变换类别；
- 建立策略注册与选择机制；
- 根据模型类型或输入类型匹配变换；
- 处理变换的前置条件与适用边界；
- 为不同任务类型保留扩展位。

### I4：模型层 MR 生成器第一版

目标：把结构分析结果与变换策略结合，生成模型层 MR。

本任务需要完成：

- 设计模型层 MR 生成主流程；
- 根据结构模式选择候选关系模板；
- 根据输入/任务类型选择增强或扰动策略；
- 产出可执行的模型层 MR 记录；
- 与统一 repository 对接。

### I5：模型层 Oracle 与验证器

目标：让模型层测试具备可执行、可解释的判定逻辑。

本任务需要完成：

- 梳理模型层可用的结果判定方式；
- 支持标签一致、排名稳定、嵌入相近、loss 约束等基础验证形式；
- 处理不同模型任务下的判定差异；
- 输出失败原因摘要；
- 与现有结果存储和证据收集流程打通。

### I6：模型层测试执行器与批量运行入口

目标：让模型层 MR 不只是能生成，还能批量运行。

本任务需要完成：

- 扩展现有 test 主链支持模型层对象；
- 建立模型层输入准备与调度逻辑；
- 支持单模型和批量模型测试；
- 记录运行结果、失败样本和统计摘要；
- 保持与统一 IR / run manifest 一致。

### I7：模型层失败案例与报告输出

目标：让模型层测试结果能够直接服务论文实验与案例分析。

本任务需要完成：

- 输出模型层失败案例摘要；
- 形成模型级报告视图；
- 支持统计模型层 MR 数量、通过率、失败率和案例数；
- 与 evidence pack 或其扩展形式对齐；
- 为实验组织器提供数据接口。

### I8：模型层测试与回归加固

目标：确保模型层能力不是演示脚本，而是可持续运行能力。

本任务需要完成：

- 为模型分析器、策略库、生成器、验证器和执行器补齐单元测试；
- 建立至少一条模型层集成测试主链；
- 验证模型层结果能被实验与报告模块消费；
- 补充使用说明和已知限制。

本阶段建议优先新增或调整以下区域：

## 6. 可运行成果

本阶段结束后，系统至少应能做到：

1. 识别并加载一批代表性模型；
2. 自动生成一批模型层 MR；
3. 批量执行模型层测试并持久化结果；
4. 输出模型层失败案例与统计摘要；
5. 将模型层结果纳入实验组织器。

---

## 7. 验收标准

本阶段验收重点是“模型层是否形成真实闭环”：

- 至少一批模型基准对象可在统一入口下运行；
- 模型结构分析结果可驱动 MR 生成；
- 模型层测试结果可落库、可统计、可报告；
- 至少有一条完整的模型层 case 可以复现；
- 新增测试覆盖模型层核心模块与主链。

---

## 8. 与论文章节的对应关系

本阶段主要支撑以下论文内容：

- 第 3.3 节：模型层蜕变关系自动生成方法；
- 第 4.2.2 节：模型层生成引擎实现；
- 第 5.3 节：分层缺陷检测能力评估中的模型层部分。

---

## 9. 开发完成标志

当以下条件同时满足时，可认为 Phase I 完成：

- 模型层对象、分析、生成、验证、执行全部进入主链；
- 模型层不再只是论文描述，而是有真实结果输出；
- 模型层结果可进入实验和报告体系；
- 系统具备继续扩展更多模型的稳定入口。
